5 Types of branches

  • Master branch-the master branch is the production branch. The code living in the master branch is what the running product has.
  • Release branch-the release branch is pulled from the master branch. The release branch is the branch where the code is pushed to when ready for release. This is where the code lives before it makes it to production. Additional unit testing is run on code on this branch.
  • Hotfix branch-the hotfix branch is pulled from master branch. The purpose of the hotfix branch is to fix issues in production in a quick manner without a sprint. This branch is only used it's an occasion of a urgent issue.
  • Develop branch-the develop branch is pulled from the master branch. The develop branch is the branch where any code that has been developed in the current sprint lives. This is code meant for the next release but is not ready for release yet.
  • Feature branch-the feature branch is pulled from the development branch. The feature branch is where the development of the code happens. A feature branch is used per feature.

A ticket process

  1. First you will need your ticket for the feature that is being worked on.
  2. Checkout master branch. By default you should be on master branch unless if switched to a different branch.
  3. Do a git fetch and git pull in order to pull exact code from master into local.
  4. Checkout develop branch.
  5. Do a git fetch and git pull in order to ensure code on develop branch is up to date with code on master.
  6. Checkout a new branch. The newly created branch is considered a feature branch and will branch out out from develop.
  7. New branch will have the code from develop up to this point
  8. Coding can not begin on feature branch.
  9. Once feature has been completed and ready to be tested, merge feature branch into development branch.
  10. If no merge conflicts occured, code should have successfully joined the code in develop./li>
  11. Testing will occur on develop branch.
  12. Once feature is verified to be ready for production, develop branch is merged into release branch.
  13. Once code is on release branch, additional testing is done to verify code is absolutely ready to be in production.
  14. Now release branch will be merged into master branch.
  15. Master branch will now have the new code.
  16. If any P1 issues occur that NEED to be fixed asap, a new branch is pulled from master branch in order to crate a hotfix branch and fix the code.
  17. When code on hotfix branch has been tested and accepted, hotfix branch will be merged into master branch.
